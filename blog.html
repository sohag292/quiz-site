1. What is the differents between forEach, find, map, & filter in javascript?

Ans: map() executes the same code on every element in an array and returns a new array with the updated elements. forEach() is used to execute the same code on every element in an array but does not change the array and it returns undefined. filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria. find() returns first element of the array that satisfies the condition specified in the callback function.

2. What is the differents between localStorage, sessionStorage & cookies?

Ans: Local storage Local storage stores a larger amount of data on the client's computer in a key-value pair format and has no expiration date. Data is never transferred to the server and is accessible via JavaScript and HTML5. Its Capacity: 10MB. Session storage stores a larger amount of data on the client's computer only for the current session, expiring the data on tab close. Data is never transferred to the server and is accessible client-side from the same tab. Its Capacity: 5MB. Cookies store small amounts of data that has to be sent back to the server with subsequent requests and their expiration can be set from either server or client. They are primarily used for server-side reading. Its Capacity: 4KB

The call stack is responsible for keeping track of all the operations in line to be executed. Whenever a function is finished, it is popped from the stack.The event queue is responsible for sending new functions to the stack for processing. 5. How Does the chooseQuiz Function works?

Ans: The chooseQuiz function is used to update the user's answer to a quiz. It does this by checking if the quiz has already been answered before, and if it has, it updates the answer in the answers array.
